# Code of Conduct

OpenSyria is built for public benefit. We want project spaces to be respectful, careful, and useful for people contributing from different backgrounds, languages, locations, and levels of technical experience.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent, but accept correction.
- Discuss ideas, data, sources, design, accessibility, content, and implementation choices without personal attacks.
- Keep feedback specific and actionable.
- Respect project scope, licensing rules, and safety boundaries.
- Avoid sharing private, personal, sensitive, restricted, military, or surveillance-related information.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing private information without permission.
- Sharing secrets, credentials, private infrastructure details, or personal data.
- Pressuring maintainers to accept out-of-scope work.
- Introducing or requesting content or datasets that could endanger people or violate source licenses.
- Repeatedly ignoring maintainer decisions or repository contribution rules.

## Enforcement

Maintainers may edit, hide, lock, or remove comments, issues, discussions, or pull requests that violate this code of conduct. Maintainers may also block users from participating in the repository when needed.

Security issues should be reported privately through [SECURITY.md](SECURITY.md), not through public issues.

## Scope

This code of conduct applies to this repository and related OpenSyria project spaces.
